# 31. Next Permutation

给定一个数组表示一个 permutation，要找到 **比当前 permutation 稍大** 的下一个排列lexicographically larger 如果这个排列已经是最大的，就返回最小排列（排序成升序）。算法必须是 **就地（in-place）** 完成。

## Approach

下一个更大的 permutation 必须从 **右侧** 调整，因为右边是“最低位”。

比如：`[1, 2, 3, 6, 5, 4]`

我们希望变成稍微大一点的排列。

Step 1：从右往左找到第一个下降点（pivot）
    
找到 **nums[i] < nums[i+1]** 的那个位置。这是因为从右边开始第一次出现“往下掉”的地方，才能通过交换让排列变大。
    
例：`1 2 3 6 5 4`  下降点是 `3 < 6` → pivot = index 2

Step 2：从右往左找到第一个比 pivot 大的数
在 pivot 右边找到 **最小的大于 nums[pivot]** 的元素。

因为要让结果“刚刚好变大”。

例：  
右侧区间 `[6 5 4]` 中  
比 `3` 大且最小的是 `4`

交换：

`1 2 4 6 5 3`

Step 3：把 pivot 右边的部分全部反转
右边那段原本是 **降序** 的，反转后变成最小升序。

`1 2 4 6 5 3` → reverse 右半部分 →  
`1 2 4 3 5 6`

这就是下一个更大的排列。


```java
class Solution {
    public void nextPermutation(int[] nums) {
        int n = nums.length;
        int i = n - 2;

        // Step 1: find pivot
        while (i >= 0 && nums[i] >= nums[i + 1]) {
            i--;
        }

        // Step 2: find the next larger number and swap
        if (i >= 0) {
            int j = n - 1;
            while (j > i && nums[j] <= nums[i]) {
                j--;
            }
            swap(nums, i, j);
        }

        // Step 3: reverse the suffix
        reverse(nums, i + 1, n - 1);
    }

    private void reverse(int[] nums, int left, int right) {
        while (left < right) {
            swap(nums, left++, right--);
        }
    }

    private void swap(int[] nums, int a, int b) {
        int temp = nums[a];
        nums[a] = nums[b];
        nums[b] = temp;
    }
}
```

- time: O(n)
- space: O(1)
